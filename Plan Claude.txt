# Plan de Desarrollo Completo: Radares-Profesion

## Contexto
El proyecto Radares-Profesion necesita evolucionarse de "prototipo funcional" a "producción-ready". Actualmente tiene diseño moderno pero carece de validación de datos, funcionalidades clave, y automatización CI/CD. El usuario quiere desarrollar TODO en el **orden más favorable técnicamente**.

---

## 🎯 Orden Recomendado (Fundamentación Técnica)

**Por qué este orden:**
1. **Calidad primero** → Base sólida previene bugs cascada
2. **Datos segundo** → Necesarios para probar funcionalidad
3. **Funcionalidad tercero** → Requiere datos + validación robusta
4. **Diseño cuarto** → Polido visual sobre base estable
5. **Infraestructura último** → CI/CD automatiza trabajo anterior

### 4.1 Dark/Light Mode Toggle Persistente
**Archivos: `styles.css`, template en `generate.js`, `index.html` template**

```javascript
// Agregar script inline en <head> (antes de CSS):
const theme = localStorage.getItem('theme') || 'auto';
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
const isDark = theme === 'auto' ? prefersDark : theme === 'dark';
document.documentElement.classList.toggle('dark-mode', isDark);

// Toggle button en header (☀️/🌙)
- Click → cicla: auto → light → dark → auto
- Almacena en localStorage
- Transición suave 0.3s
```

## FASE 5: INFRAESTRUCTURA (Automatización)
**Duración estimada: 6-8 horas**

### 5.1 Crear GitHub Actions CI/CD
**Archivo: crear `.github/workflows/build-deploy.yml`**

```yaml
name: Build & Deploy
on:
  push:
    branches: [main, claude/**]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm ci
      - run: node validate.js        # Validar professions.json
      - run: node generate.js        # Generar radares
      - run: npm test                # Tests básicos
      - run: npm run screenshot      # Screenshots automáticos (opcional)
      - name: Deploy to GitHub Pages
        if: github.ref == 'refs/heads/main'
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: .
```

### 5.2 Mejorar serve.js (caching + compresión)

## ⚙️ Decisiones de Diseño

1. **localStorage sobre servidor** - Favoritos y historial locales, sin backend
2. **Validación en build time** - Previene deployes con datos inválidos
3. **CSS variables dinámicas** - Colores por profesión sin hardcode
4. **GitHub Pages primario, Cloudflare secundario** - Redundancia y CDN
5. **Tests simples sobre complejos** - MVP de testing, no overkill
